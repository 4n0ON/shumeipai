最终还是抵不住诱惑败了一个Raspberry Pi回来，折腾了一两天这玩意就是性能差点其他方面还真不错。话说树莓派自带的raspbian是针对armv6架构CPU进行优化的debian发行版，支持硬浮点的armhf的性能多少会比armel好一些。

查了一下资料，据说这是到2012-9-4时依然是一个没有解决的bug，不过除了等官方修复或者直接改用openelec、XBian之类的专用发行版之外还可以用下面的方法（其实就是改源和添加公钥）：

11. 向 /etc/apt/sources.list里添加  deb ftp://ftp.deb-multimedia.org wheezy main non-free
22. 执行 sudo apt-get update, 这时候会返回一些因为没有公钥无法验证的错误，如果你不是很有洁癖可以直接跳到最后一步直接安装xbmc
3//针对有洁癖的人安装deb-multimedia-keyring
43. sudo apt-get install deb-multimedia-keyring
54. sudo apt-get update
6//洁癖结束
75. 现在可以通过sudo apt-get install xbmc 来安装XBMC啦
